## 1. 프로젝트 설명

Sleepy Sheep은 HTML, CSS, JavaScript ES Modules, Canvas API, localStorage를 중심으로 만든 수면 기록형 힐링 웹앱이다. 사용자는 양 캐릭터 드리미를 돌보면서 수면 기록, 꿈일기, 고민 상담, ASMR, 출석 보상, 양털 수확, 공방 꾸미기 기능을 사용할 수 있다.

핵심 루프는 다음과 같다.

1. 홈에서 드리미 상태를 확인한다.
2. 수면을 기록하거나 모닝콜을 사용한다.
3. 출석/수면/상호작용 보상으로 양털과 컬렉션을 얻는다.
4. 양털이 충분히 자라면 양털깎기 미니게임을 플레이한다.
5. 얻은 양털로 공방 아이템을 구매하고 방을 꾸민다.
6. 갤러리에서 출석 보상 이미지와 스토리를 다시 본다.

현재 구조는 단일 프론트엔드 정적 사이트에 가깝다. 서버 없이 GitHub Pages에서 동작하며, 데이터는 사용자 브라우저의 localStorage에 저장된다. 이 구조는 배포가 쉽고 초보자 프로젝트에 잘 맞지만, 브라우저/기기별 저장소 초기화나 캐시 문제에 민감하다.

### 3.2 설정 화면

결과: 대부분 정상, 1분 테스트 문제 발견 후 수정

확인 내용:

- 설정 화면 로딩 성공
- 앱 버전 `SleepySheep v1.92.0` 표시 확인
- `지금 테스트`, `1분 뒤 테스트` 버튼 표시 확인

- 모닝콜 종료 버튼 클릭 시 모달 닫힘 확인

발견 문제:

- `1분 뒤 테스트` 클릭 시 토스트는 정상 표시됐다.
- 그러나 설정 화면에 계속 머물러 있으면 1분 후에도 모닝콜 모달이 뜨지 않았다.

원인:

- 실제 모닝콜 스케줄러는 홈 화면 중심으로 동작한다.
- 설정 화면의 테스트 버튼은 스케줄만 저장하고, 설정 화면 자체에서 알람 UI를 띄우지는 않았다.
- 브라우저 알림 권한이 없거나 서비스 워커 알림이 즉시 동작하지 않는 환경에서는 테스트가 실패한 것처럼 보일 수 있다.

수정:

- `pages/settings.html`의 `1분 뒤 테스트` 버튼에 60초 타이머를 추가했다.
- 시간이 지나면 `../index.html?morningcall=1`로 이동해 실제 모닝콜 테스트 모달을 띄운다.
- 여러 번 누르면 이전 타이머를 지우고 새 타이머만 유지한다.

## 4. 남은 문제 및 개선 제안

### 4.1 GitHub Pages 실제 경로가 `PBL1/PBL1`로 보임

상태: 기능은 동작하지만 구조적으로 어색함

현재 사용자가 접속하는 링크:

```text
https://cbr20266112.github.io/PBL1/index.html
```

실제 앱이 열리는 경로:

```text
https://cbr20266112.github.io/PBL1/PBL1/index.html
```

원인:

- GitHub Pages는 저장소 루트를 `https://cbr20266112.github.io/PBL1/`로 제공한다.
- 현재 저장소 루트에는 별도 `index.html`이 있고, 실제 앱은 그 아래 `PBL1/` 폴더 안에 있다.
- 루트 `index.html`이 `PBL1/index.html`로 리다이렉트하므로 URL에 `PBL1`이 한 번 더 붙는다.

영향:

- 현재 기능은 정상 동작한다.
- 하지만 사용자가 주소를 볼 때 경로가 길고 어색하다.
- 상대 경로, PWA, 서비스 워커, 공유 링크를 다룰 때 혼동 가능성이 있다.

권장 방향:

1. 단기: 현재 구조 유지. 사용자가 준 링크는 정상 접속되므로 급한 문제는 아니다.
2. 중기: GitHub Pages 배포 루트를 실제 앱 폴더로 맞추거나, 앱 파일을 저장소 루트로 이동하는 구조 정리를 검토한다.
3. 구조를 옮길 경우 상대 경로, 서비스 워커 scope, manifest, 아이콘 경로를 함께 점검해야 한다.

### 4.2 루트 `index.html`의 한글이 깨져 있음

상태: 사용자에게 거의 보이지 않지만 품질 문제

확인 내용:

- 저장소 루트의 `index.html`에는 UTF-8 선언이 있지만 본문 일부 한글이 깨져 있다.
- 바로 리다이렉트되므로 일반 사용자는 거의 못 보지만, 네트워크가 느리거나 리다이렉트가 막히면 깨진 문구가 보일 수 있다.

권장 방향:

- 루트 `index.html` 본문과 title을 정상 한글로 다시 저장한다.
- 다만 이 파일은 현재 작업 루트 `PBL1/` 밖에 있으므로 별도 권한과 커밋 범위 조정이 필요하다.

### 4.3 온보딩 데스크톱 여백

상태: 기능 문제는 아니지만 시각 개선 가능

현상:

- 데스크톱에서 오프닝 이미지가 모바일 프레임 안에 들어가며 위아래 검은 여백이 크게 보인다.

권장 방향:

- 이미지 컨테이너 배경을 검정 대신 앱 배경색/그라데이션과 맞춘다.
- 또는 이미지 `object-fit`/프레임 비율을 조정해 검은 여백을 줄인다.

### 4.4 모바일 자동화 검증 제한

상태: 도구 제한

내용:

- 브라우저 자동화 도구에서 viewport를 390x844로 바꾸려 했지만 실제 `innerWidth`가 1280으로 유지됐다.
- 따라서 이번 자동 검수는 기본 데스크톱 폭 중심으로 진행했다.

권장 방향:

- 실제 휴대폰 또는 Chrome DevTools 모바일 모드에서 홈, 공방, 미니게임, 설정 화면을 한 번 더 눈으로 확인한다.

## 5. 설계 분석

### 5.1 좋은 점

- 파일이 기능별로 비교적 잘 나뉘어 있다.
- `js/storage.js`, `js/constants.js`, `js/morningcall.js`, `js/attendance.js`, `js/sleep.js`처럼 도메인별 책임이 분리되어 있다.
- 정적 배포에 적합하고, 서버 없이도 대부분 기능이 동작한다.
- UI는 전체적으로 귀엽고 일관된 수면/양 콘셉트를 유지한다.
- 미니게임은 Canvas 레이어를 분리해 바디/털/커서를 따로 관리하는 방식이라 확장성이 좋다.

### 5.2 주의할 점

- localStorage 기반 앱이므로 데이터 구조가 바뀔 때 마이그레이션 전략이 필요하다.
- 날짜 처리는 반드시 로컬 날짜 기준으로 유지해야 한다. UTC 날짜 문자열을 쓰면 한국 시간 기준 새벽에 날짜가 어긋날 수 있다.
- 모닝콜은 브라우저 탭 상태, 알림 권한, 서비스 워커 상태에 영향을 많이 받는다. 테스트 버튼처럼 사용자가 즉시 확인하는 기능은 화면 이동이나 모달 표시까지 확실히 연결하는 편이 좋다.
- GitHub Pages 경로가 현재 `PBL1/PBL1` 구조라 PWA 경로와 상대 경로를 계속 조심해야 한다.

### 5.3 앞으로의 추천 작업 순서

1. 루트 `index.html` 한글 깨짐 수정
2. GitHub Pages 배포 경로 구조 정리 여부 결정
3. 모바일 실기기 화면 검수
4. 온보딩 검은 여백 개선
5. 모닝콜 브라우저 알림 권한 흐름 UX 개선
6. localStorage 데이터 버전/마이그레이션 설계 추가
7. 갤러리 해금 데이터가 실제 출석 보상과 잘 이어지는지 장기 시나리오 테스트

## 6. 최종 판단

v1.92.0 배포본은 주요 기능이 정상 동작한다. 홈, 설정, 모닝콜 즉시 테스트, 갤러리, 공방, 양털깎기 미니게임, 수면 페이지, 홈 ASMR은 큰 오류 없이 확인됐다.

실제 사용 중 바로 문제가 될 수 있었던 항목은 `1분 뒤 테스트` 버튼이었다. 설정 화면에 머무르면 알람이 뜨지 않는 문제가 확인되어, 설정 화면에서도 60초 후 홈 모닝콜 테스트로 이동하도록 수정했다.

남은 개선점은 대부분 구조/품질 이슈다. 특히 `PBL1/PBL1` 경로와 루트 `index.html` 한글 깨짐은 기능을 막지는 않지만, 배포 품질을 높이려면 다음 단계에서 정리하는 것이 좋다.
