# v1.92.7 Quality Stabilization Plan

Date: 2026-07-16

Status: planning only; no implementation changes made.

Scope: This plan combines the findings from [docs/workshop-shop-audit-report.md](workshop-shop-audit-report.md) and [docs/asmr-audit-report.md](asmr-audit-report.md) into a release-focused stabilization plan for v1.92.7.

Goal: Fix the most user-visible quality issues before adding new features.

---

## 1. Critical issues

These issues should be addressed first because they directly affect trust, clarity, and immediate user experience.

### 1.1 Workshop data integrity

The workshop catalog is structurally wired correctly, but the content quality is still fragile because the system depends on item ID → image filename matching and visual interpretation.

Risks:
- wrong image shown for an item name
- item displayed under the wrong category
- item name does not match the visible asset
- placeholder or missing asset fallback makes the shop feel incomplete

### 1.2 Friend page localization

Several friend-page strings are still not fully localized and will fall back to English when the Korean locale is active.

High-risk strings include:
- add-friend input placeholder
- search button
- status/time labels
- sleeping state labels
- room title and stats text
- locked-room message

### 1.3 Day-of-week localization

The weekday labels used by calendar/sleep views are still not fully localized in the Korean bundle, which makes the app feel inconsistent in the calendar and sleep history screens.

### 1.4 ASMR localization

A large number of ASMR card names and descriptions still fall back to English because the corresponding i18n keys are missing.

This is highly visible because ASMR is one of the app’s core experiences.

### 1.5 ASMR content quality

The current ASMR catalog contains multiple overlapping entries that are difficult to distinguish by ear and several entries whose descriptions are more polished than the generated sound.

This creates confusion and weakens the perceived quality of the ASMR experience.

### 2.3 ASMR catalog cleanup without adding new ambience

#### Duplicate or near-duplicate entries to consolidate

The current catalog should be reduced first rather than expanded.

Duplicates / near-duplicates:
- `rain` + `preset_rainy`
- `ocean` + `preset_ocean`
- `river` + `water`
- `fire` + `preset_cozy`
- `cottage` + `preset_cottage`
- `brown` + `preset_deep`
- `zen_garden` + `preset_zen`
- `koto` + `preset_traditional`
- `pencil` + `brush` + `preset_study`

#### Weak entries to rework or remove

- `water`: description is stronger than the actual sound; should be reworked or replaced
- `preset_study`: too generic and not clearly “bookshelf / library” in feel
- `preset_starry`: too similar to `ranch` and not distinct enough
- generic white noise / brown noise entries: should be demoted or removed from the primary catalog

#### Recommended ASMR direction for v1.92.7

Do not add new ambience scenes yet. Instead:
- keep the strongest current entries
- remove or merge the weakest duplicates
- align names/descriptions more tightly to the actual generated audio

This keeps the release focused on stability and clarity rather than feature expansion.

## 3. Nice-to-have improvements

These are useful but should not block the release if time is tight.

1. Add a simple item review checklist for shop content before release.
2. Add an explicit item metadata table so the app can validate item name/category/asset consistency programmatically.
3. Add a localization regression checklist for workshop, friends, calendar, and ASMR screens.
4. Add a short “catalog quality review” pass for ASMR descriptions before each release.

## 4. Recommended next release scope

### v1.92.7 scope

Include only the stabilization work below:
- workshop asset/name/category review
- friend page localization fixes
- weekday localization fixes
- ASMR localization fixes

### Out of scope for v1.92.7

- adding new ambience scenes
- introducing a new ASMR catalog structure
- major redesign of the workshop/shop UI

### Release recommendation

v1.92.7 should be treated as a quality-and-stability release. The priority is to make the existing experience feel coherent, localized, and trustworthy before introducing new features.
